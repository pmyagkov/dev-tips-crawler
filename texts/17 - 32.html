17. DevTools: Quick-edit the HTML Element tag (https://umaar.com/dev-tips/17-quick-edit-element/)

You can double click on the opening tag name of a node in the Elements Panel to edit it. When you do, the closing tag is automatically updated with your change.﻿

17. Быстрое редактирование названия HTML-тега в панели "Elements"

Если вам нужно поменять название DOM-узла в панели "Elements", дважды кликните на его открывающий тег и введите новое имя. При этом закрывающий тег элемента автоматически будет соответствующе изменен.

18. DevTools: Alt + Click to expand all child nodes

You can use Alt + Click in the Elements Panel to expand all child nodes. Just click on the small arrow to the left of the DOM node while holding Alt.

18. Разворачивание всех дочерних узлов выбранного элемента

Вы можете развернуть все вложенные узлы выбранного элемента, удерживая клавишу Alt и кликнув по стрелке рядом с открывающим тегом элемента.

19. DevTools: Toggle the DevTools dock state with a keyboard shortcut (https://umaar.com/dev-tips/19-dock-undock-shortcut/)

Use Cmd + Shift + D to toggle the DevTools dock state between the last two recently used states. Dock options are:

  • Dock to right
  • Dock to bottom
  • Undock DevTools

19. Переключение состояния вкладки DevTools при помощи горячих клавиш

Переключайте два последних используемых режима отображения панели DevTools при помощи Cmd + Shift + D. Панель поддерживает следующие опции показа:

  • Прикреплена к правому краю окна браузера
  • Прикреплена к нижнему краю окна браузера

29. DevTools: 5 tricks to use in the Console Panel (https://umaar.com/dev-tips/29-five-console-tricks/)

Use the inspect() command to jump straight to a passed in DOM node

inspect($('p'))

The copy() command to copy text to your clipboard

copy(Object.keys(window))

    // stores ["top", "window", "location", "external"... and so on

Styled console output:

console.log('%cHello world', 'font-size:40px;color:#fff;text-shadow:0 1px 0 #ccc,0 2px 0 #c9c9c9,0 3px 0 #bbb,0 4px 0 #b9b9b9,0 5px 0 #aaa,0 6px 1px rgba(0,0,0,.1),0 0 5px rgba(0,0,0,.1),0 1px 3px rgba(0,0,0,.3),0 3px 5px rgba(0,0,0,.2),0 5px 10px rgba(0,0,0,.25),0 10px 10px rgba(0,0,0,.2),0 20px 20px rgba(0,0,0,.15);');

Get the values of an object

    one: 1,
    two: 2,
    three: 3
    })

    // logs [1, 2, 3]

Clear the console

Cmd + K (Ctrl + L on Windows)

29. 5 интересных возможности панели "Console"

  1. Выбирайте конкретный DOM-узел в панели "Elements", передав его в функцию inspect(node);

  2. Копируйте текст при помощи команды copy(text):

    copy(Object.keys(window)) // скопирует все свойства объекта window ("top", "window", "location" и т.д.)

  3. Стилизуйте консольный вывод:

    console.log("%cHello world", "font-size:40px;color:#fff;text-shadow:0 1px 0 #ccc,0 2px 0 #c9c9c9,0 3px 0 #bbb,0 4px 0 #b9b9b9,0 5px 0 #aaa,0 6px 1px rgba(0,0,0,.1),0 0 5px rgba(0,0,0,.1),0 1px 3px rgba(0,0,0,.3),0 3px 5px rgba(0,0,0,.2),0 5px 10px rgba(0,0,0,.25),0 10px 10px rgba(0,0,0,.2),0 20px 20px rgba(0,0,0,.15);")

  4. Получайте массив значений объекта:

    values({ one: 1, two: 2, three: 3 }); // выведет [1, 2, 3]

  5. Очищайте консоль при помощи горячих клавиш Cmd + K.

30. DevTools: Console Panel autocomplete with properties (bracket or dot notation) (https://umaar.com/dev-tips/30-console-autocomplete/)

Autocomplete in the Console Panel not only works with regular dot notation (e.g. window.onloa → window.onload), but also with square bracket notation e.g. window['onloa → window['onload'].

Even if you have an array, you get autocomplete for the index e.g. arr[0 → arr[0].

30. Автодополнение свойств и методов объекта в панели "Console"

Автодополнение в консоли работает не только при использовании точки (window.onlo → window.onload), но и при использовании квадратных скобок (window['onloa → window['onload']).

Также вы увидите его и при отладке массивов (arr[0 → arr[0]).

31. Просмотр и отладка обработчиков событий

Используйте функцию getEventListeners(node) в консоли для получения всех зарегистрированных обработчиков событий для переданного DOM-элемента. Кроме этого есть конструкция debug(fn), останавливающая исполнение программы, в момент вызова переданной функции.

32. DevTools: Automatically pause on any exception (https://umaar.com/dev-tips/32-pause-on-exception/)

In DevTools, you can automatically pause on any exception (especially useful for uncaught exceptions). Just enable 'Pause on exceptions' within the Sources Panel. Optionally, you can also pause on caught exceptions.

This means the debugger will pause before that red message appears in the Console and you get a chance to inspect what may have gone wrong.

32

32. Автоматическая остановка выполнения программы при возникновении любых исключений

Вы можете останавить поток исполнения при возникновении любого исключения (это особенно полезно в случае непойманных исключений). Для этого включите опцию "Pause on exceptions" в панели "Sources". Кроме этого вы можете поймать все обработанные исключения, установив галочку "Pause on caught exceptions".

При использовании этой функциональности у вас появится возможность выявить проблему в коде до фактического возникновения ошибки.
